1. 실습 준비 

이번 실습에서는 블루프린트를 이용해 움직이는 액터를 만들어 보겠습니다. 지난 실습에 사용했던 프로젝트를 그대로 이어서 사용합니다.

새로운 레벨을 만들어서 실습을 진행하겠습니다.
이름은 MovingActorMap 으로 합니다.

2. 액터 배치 및 머티리얼 적용

이제 이 레벨 위에 트로피로 사용할 프롭을 배치해 보겠습니다. 스타터 콘텐츠의 Props 폴더로 이동합니다. SM_Statue 등 원하는 스태틱 메쉬를 찾아 레벨 위쪽으로 드래그해서 배치합니다.

이 트로피를 금빛으로 바꿔보겠습니다. Materials 폴더로 이동해서 Gold 머티리얼을 선택한 뒤, 액터 위로 드래그해서 적용해 줍니다. 그러면 반짝이는 금빛 트로피가 완성됩니다.

이 트로피를 위아래로 출렁거리게 만드는 것이 이번 실습의 최종 목표입니다. 우선은 트로피를 위로 조금씩 계속 올라가게 만드는 것부터 시작해 보겠습니다.

3. 블루프린트 클래스 생성 및 에디터 구성

방금 배치한 트로피를 선택합니다. 오른쪽 디테일 패널을 보면 노드와 와이어가 연결된 모양의 아이콘이 있습니다. 이 버튼은 해당 액터를 블루프린트 클래스로 전환해 주는 버튼입니다. 클릭해 보겠습니다.

새 블루프린트를 생성하는 패널이 뜹니다. 여러 옵션이 나오는데, 이는 추후에 자세히 살펴볼 예정입니다. 만들어질 블루프린트의 이름은 BP_GoldTrophy로 지정하겠습니다. 저장 경로는 방금 만들었던 MovingActor 폴더로 지정한 뒤, 생성 버튼을 누릅니다.

블루프린트가 생성되면 새 창이 열립니다. 이 창이 바로 블루프린트 에디터입니다. 이 에디터는 보통 독립된 창으로 열리는데, 지금처럼 모니터 한 대로 작업할 때는 이 탭을 메인 에디터 창에 도킹시켜서 사용하는 것이 편리합니다. 탭을 클릭한 채로 드래그해서 원하는 위치에 붙여 주세요.

참고로 탭 이름 옆에 별 모양 표시가 보인다면 아직 저장되지 않았다는 뜻입니다. Ctrl+S를 눌러 수시로 저장하는 습관을 들이는 것이 좋습니다.

4. 이벤트 그래프 조작 및 노드 연결

블루프린트 에디터의 구성을 살펴보겠습니다. 먼저 뷰포트 탭은 블루프린트의 외형을 입체적으로 보여줍니다. 실제 핵심 로직을 작성하는 작업 공간은 세 번째 탭인 이벤트 그래프입니다. 이벤트 그래프로 이동해 주세요.

여기서의 조작 방법은 간단합니다. 마우스 휠 스크롤로 확대와 축소를 할 수 있고, 마우스 오른쪽 버튼을 누른 채 드래그하면 화면을 상하좌우로 이동할 수 있습니다. 노드는 마우스 왼쪽 버튼으로 선택해서 자유롭게 위치를 옮길 수 있습니다.

그래프를 보면 현재 세 개의 노드가 기본으로 배치되어 있습니다. 이 중 이벤트 틱 노드를 사용하겠습니다. 이 노드는 게임이 시작되면 언리얼 엔진이 매 프레임마다, 예를 들어 초당 60회씩 액터에게 끊임없이 신호를 보내는 이벤트입니다. 트로피를 움직이려면 이 이벤트 틱이 발생할 때마다 새로운 위치를 설정해 주면 되겠죠?

액터의 위치를 설정하는 함수를 찾아 연결해 보겠습니다. 빈 공간에서 마우스 오른쪽 버튼을 클릭합니다. 컨텍스트 메뉴가 뜨면 검색창에 Set Actor Location을 입력하고, 이름이 정확히 일치하는 함수를 선택합니다.

이 함수는 말 그대로 액터의 위치를 결정하는 함수입니다. 타깃 항목이 셀프로 되어 있는데, 이는 자기 자신, 즉 이 트로피 액터를 의미합니다. New Location 입력 핀에서는 새로 이동할 좌표를 지정할 수 있습니다. 이제 두 노드를 이어주겠습니다. 이벤트 틱의 하얀색 실행 핀을 클릭한 채 드래그하여 Set Actor Location의 실행 핀에 연결해 줍니다.

5. 현재 위치에 이동 값 더하기

이 상태로 게임을 실행하면 어떻게 될까요? 매 틱마다 실행되지만, New Location의 값이 기본값인 0, 0, 0으로 고정되어 있어서 트로피가 맵의 원점으로 순간이동해 버릴 것입니다.

우리가 원하는 결과는 이것이 아닙니다. 현재 위치에서 매 프레임마다 위쪽으로 1센티미터씩 이동하는 것입니다. 방법은 간단합니다. 현재 위치를 가져와서, 거기에 1센티미터를 더하고, 그 값을 새 위치로 설정하는 것입니다.

그대로 만들어 보겠습니다. 다시 빈 공간에 마우스 오른쪽 버튼을 클릭하고 Get Actor Location을 검색해 생성합니다. 이 함수는 골드 트로피의 현재 위치를 가져오는 역할을 합니다.

리턴 밸류 핀 위에 마우스를 올려보면 벡터 타입 구조체인 것을 확인할 수 있습니다. 벡터는 X, Y, Z 세 개의 값으로 이루어져 있습니다. 리턴 밸류 핀을 드래그하여 빈 곳에 놓은 뒤, 검색창에 플러스 기호를 입력합니다. 연산 노드 중에서 Add 노드를 선택합니다.

우리는 위로만 움직일 것이므로, Z축 값 입력창에만 1을 입력해 줍니다. 마지막으로, 이 더하기 노드의 결과 핀을 Set Actor Location 노드의 New Location 입력 핀에 연결해 줍니다. 연결이 끝났으면 저장합니다.

6. 컴파일과 저장, 그리고 모빌리티 에러 해결

노드 연결이 끝났다면 좌측 상단의 컴파일 버튼을 누릅니다. 아주 순식간에 완료되죠? C++ 같은 프로그래밍 언어의 컴파일과는 달리, 블루프린트의 컴파일은 우리가 시각적으로 그린 그래프를 엔진이 내부적으로 빠르게 처리할 수 있는 형태로 바꾸는 과정이라 시간이 오래 걸리지 않습니다. 하지만 그래프를 수정했다면 반드시 컴파일과 저장을 해야만 실제 게임에 반영된다는 점을 잊지 마세요.

컴파일을 마쳤으니 메인 레벨 에디터로 돌아가 플레이 버튼을 눌러보겠습니다. 실제로 트로피가 위로 올라가는지 확인해 볼까요?

어라, 올라가지 않네요. 플레이를 종료해 보면 뷰포트 우측 하단에 에러 메시지가 잔뜩 뜬 것을 볼 수 있습니다. 사실 이 에러는 여러분이 실습할 때 흔히 겪을 수 있는 상황을 보여드리기 위해 일부러 유도한 것입니다. 메시지를 해석해 보면, 이 액터를 움직이려면 모빌리티 설정을 무버블로 변경해야 한다는 뜻입니다.

문제를 해결해 보겠습니다. 트로피 액터를 선택하고 오른쪽 디테일 패널로 이동합니다. 중간쯤에 모빌리티라는 속성이 보입니다. 액터가 게임 중에 움직일 수 있는 상태인지를 나타내는 옵션입니다. 현재 기본값이 스태틱으로 설정되어 있습니다. 위치가 고정된 액터인데 블루프린트에서는 움직이라고 명령을 내리니, 엔진이 모순이라며 에러를 발생시킨 것입니다.

이 모빌리티 값을 무버블로 변경해 줍니다. 변경 후 컨트롤과 에스를 눌러 저장하고 다시 플레이해 보겠습니다. 이제 금빛 트로피가 하늘을 향해 스르륵 멈추지 않고 올라가는 것을 확인할 수 있습니다.

이상으로 실습을 마치도록 하겠습니다.
