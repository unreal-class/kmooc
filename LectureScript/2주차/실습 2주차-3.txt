이번 실습에서는 블루프린트의 변수 노출 기능에 대해서 알아보겠습니다.

앞의 실습에서 제작한 BP_GoldTrophy는 Blueprint Class입니다. Class는 C++에서 사용하는 Class와 같은 개념으로, 쉽게 말해 흔히 붕어빵 틀에 비유합니다. 붕어빵 틀이 있으면 그 틀을 이용해 붕어빵을 계속 찍어낼 수 있는 것과 같습니다. 우리가 지금까지 진행한 작업은 위아래로 움직일 수 있는 골드 트로피의 Class를 만든 것입니다. Class가 존재하면 이를 이용해 실제 레벨에 배치되는 Actor를 무수히 찍어낼 수 있습니다.

방법은 아주 간단합니다. 콘텐츠 브라우저에 있는 이 Class를 클릭한 채 드래그해서 레벨 위로 배치하면 됩니다.

Content Browser에서 BP_GoldTrophy를 마우스 좌클릭으로 드래그하여 Viewport 레벨 창에 내려놓습니다.
원하는 만큼 여러 개를 드래그해서 배치해 봅니다.
상단 툴바에서 Play를 눌러 실행해 봅니다.
배치한 모든 트로피들이 똑같은 높이와 속도로 움직이는 것을 볼 수 있습니다.

이렇게 만든 트로피를 기획자에게 전달했다고 가정해 보겠습니다. 기획자가 "어떤 트로피는 위아래로 작게 움직이고, 어떤 트로피는 큼직하게 움직이게 해달라"고 요청할 수 있습니다. 지금은 모든 트로피가 50cm 폭으로 고정되어 있지만, 기획자는 어떤 트로피는 10cm, 어떤 트로피는 100cm로 조절하고 싶어 할 것입니다.

이런 요청이 있을 때마다 개발자가 매번 Blueprint 에디터를 열고 코드를 수정해야 한다면 무척 비효율적이겠죠. 이때 사용하는 것이 바로 Instance Editable이라는 변수 옵션입니다.


BP_GoldTrophy 블루프린트 에디터 창으로 다시 이동합니다.

위아래 움직임의 폭을 결정하는 요소는 아까 Sin 노드 뒤에 연결했던 Multiply 노드의 50이라는 숫자입니다. 이 숫자를 변수로 만들어 주겠습니다. 왼쪽 패널에서 변수를 새로 만들어 연결해도 되지만, 더 간편한 기능이 있습니다.

Sin 노드와 연결된 Multiply 노드를 찾습니다.
숫자 50.0이 입력되어 있는 아래쪽 핀(B 핀) 위에서 마우스 우클릭을 합니다.
컨텍스트 메뉴 상단에 있는 Promote to Variable을 선택합니다.
그러면 자동으로 핀에 연결된 Float 타입의 변수가 하나 생성됩니다.
왼쪽 My Blueprint 패널의 Variables 항목에 새 변수가 추가된 것을 볼 수 있습니다. 변수 이름을  Amplitude로 변경해 줍니다.
>>>
이름을 바꾼 다음 좌상단 툴바에서 Compile 버튼을 클릭합니다.
컴파일을 완료하면 오른쪽 Details 패널 하단에 Amplitude의 기본값(Default Value)이 50.0으로 들어가 있는 것을 확인할 수 있습니다.


이제 My Blueprint 패널의 Variables 항목에 있는 Amplitude 변수를 다시 확인해 봅니다. 변수 이름 오른쪽에 눈을 감고 있는 듯한 조그만 아이콘이 보입니다. 이는 이 변수가 외부로 노출되어 있지 않다는 뜻입니다.

Amplitude 변수 오른쪽의 감겨 있는 눈 아이콘을 마우스 좌클릭으로 눌러서 눈을 뜨게 만듭니다. (아이콘이 켜진 눈 모양으로 바뀝니다.)
눈을 뜨게 만든 이 상태가 바로 Instance Editable 옵션을 활성화한 것입니다. 이제 블루프린트 외부, 즉 레벨 에디터에서 각 액터마다 이 값을 따로 편집할 수 있게 됩니다.
좌상단 툴바에서 Compile과 저장 버튼을 누릅니다.
메인 레벨 에디터 창으로 이동합니다.
>>>
레벨 뷰포트에 배치된 여러 개의 트로피 중 하나를 마우스 좌클릭으로 선택합니다.
오른쪽 Details 패널을 확인해 봅니다. 패널을 조금 아래로 내려보면 Default 섹션 아래에 방금 우리가 노출시킨 Amplitude 변수가 나타난 것을 볼 수 있습니다.
>>>
이 값을 100으로 수정합니다.
>>>
다른 트로피도 선택해서, 이 값을 각각 다르게 설정해보도록 하죠. 
이 트로피는 100.0으로, 또 다른 트로피는 200으로 각각 다르게 수정해 줍니다.
>>>
상단 툴바에서 Play를 눌러 확인해 봅니다.... 각 트로피마다 위아래로 출렁거리는 진폭이 다르게 작동하는 것을 확인할 수 있습니다.


이번에는 위아래로 왕복하는 속도, 즉 주파수도 같은 방식으로 변수화해 보겠습니다. 현재 시간 값에 2파이(6.28)가 곱해져 있어서 1초에 한 번 왕복하도록 고정되어 있습니다. 여기에 추가로 숫자를 곱할 수 있도록 핀을 늘려 변수로 만들어 보겠습니다.

BP_GoldTrophy 블루프린트 에디터 창으로 돌아옵니다.
Get Game Time in Seconds 노드 뒤에 있는 Multiply 노드를 찾습니다.
Multiply 노드 위에서 마우스 우클릭을 한 뒤, 컨텍스트 메뉴에서 Add Pin을 선택합니다.
노드 아래쪽에 입력 핀(C 핀)이 하나 더 추가된 것을 볼 수 있습니다.
>>>
새로 추가된 입력 핀 위에서 마우스 우클릭을 한 뒤, 마찬가지로 Promote to Variable을 선택합니다.
왼쪽 My Blueprint 패널에 새 변수가 생성되면 이름을 Frequency로 변경해 줍니다.
>>>
좌상단 툴바에서 Compile을 누른 뒤, 오른쪽 Details 패널에서 Frequency의 기본값(Default Value)을 1.0으로 설정해 줍니다. (1.0을 곱해야 기존의 1초 주기가 유지됩니다.)
>>>
My Blueprint 패널에서 Frequency 변수 오른쪽의 눈 아이콘을 클릭하여 인스턴스 에디터블 상태(눈을 뜬 모양)로 만들어 줍니다.
좌상단 툴바에서 다시 한번 Compile과 저장 버튼을 눌러 줍니다.

이제 최종적으로 레벨로 이동하여 값이 잘 제어되는지 확인해 보겠습니다.
메인 레벨 에디터 창으로 이동하여 배치된 트로피 중 하나를 선택합니다.
오른쪽 Details 패널의 Default 섹션을 보면 Amplitude 아래에 Frequency 변수도 새롭게 노출된 것을 볼 수 있습니다.
조금 더 빠르게 왕복시키고 싶은 트로피는 Frequency 값을 5.0 혹은 10.0으로 늘려 줍니다.
값을 변경한 뒤 상단 툴바에서 Play를 눌러 실행해 봅니다. 각 트로피들이 우리가 원하는 진폭과 속도로 제각각 다르게 움직이는 것을 확인할 수 있습니다.

이렇게 기획자나 디자이너가 에디터에서 직접 값을 바꿀 수 있도록 구조를 만들어 두는 것이 매우 중요합니다. 예전 같으면 기획자가 요청할 때마다 개발자가 소스 코드를 수정해서 빌드를 다시 넘겨주어야 했습니다. 하지만 이렇게 Instance Editable 변수로 열어두면, 기획자가 직접 원하는 느낌을 라이브로 조절할 수 있습니다.

언리얼 엔진은 개발자와 직군 간의 원활한 협업을 위해 이러한 강력한 구조를 지원하고 있으니, 앞으로 변수를 만들 때 적극적으로 활용해 보시기 바랍니다. 이상으로 이번 실습을 마치겠습니다.
