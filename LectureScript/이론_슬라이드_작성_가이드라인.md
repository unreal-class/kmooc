# 이론 슬라이드 작성 가이드라인

언리얼 엔진 강의 스크립트를 기반으로, 실습 진행 전 수강생의 사전 이해를 돕기 위한 이론 슬라이드를 작성할 때 따르는 기준입니다.

## 1. 목적

- 강의 스크립트는 이론과 실습이 혼재되어 있음
- 실습을 시작하기 전에, 그 실습을 이해하는 데 필요한 이론적 배경을 수강생이 먼저 알고 있도록 슬라이드로 정리
- 목표는 실습이 매끄럽게 진행되도록 사전 지식을 갖추는 것

## 2. 슬라이드 구성 원칙

- **한 슬라이드 = 한 주제**: 가능하면 슬라이드 한 장에는 하나의 개념/주제만 다룸. 여러 개념을 한 슬라이드에 욱여넣지 않음
- 주제를 세분화해서 필요하다면 슬라이드 수를 늘림 (개수를 아끼지 않음)

## 3. 슬라이드 형식

각 슬라이드는 다음 형식을 따름:

```
[슬라이드 제목]

[핵심 포인트 1]
[핵심 포인트 2]
[핵심 포인트 3]
[핵심 포인트 4]
```

- 첫 줄: 슬라이드 제목 (주제를 압축한 짧은 문구)
- 이후 줄: 불릿 기호 없이, 한 줄에 한 핵심 포인트씩 나열 (보통 3~5개)
- 각 포인트는 완결된 문장 또는 구 단위로 간결하게 작성

### 예시

```
블루프린트의 특징

텍스트 기반 언어(C++, 파이썬)와 달리 그림으로 로직을 구성
사각형 노드와 이를 연결하는 와이어로 구성
노드와 와이어의 집합을 블루프린트 그래프라고 부름
실행 흐름을 시각적으로 확인할 수 있어 디버깅이 용이
```

## 4. 내용 범위

- 스크립트에 등장하는 실습 내용을 이해하는 데 필요한 개념을 우선적으로 추출
- 스크립트 내용에만 한정하지 않고, 관련된 **일반 프로그래밍 언어 개념**(예: 클래스, 객체, 상속, 함수, 조건문, 이벤트 기반 프로그래밍 등)과 **언리얼 엔진 관련 배경지식**(예: 엔진 기본 클래스 구조, 콜리전 종류, 컴포넌트 시스템 등)을 함께 보강하여 내용을 풍성하게 구성
- 스크립트에 명시적으로 언급되지 않았더라도, 실습을 이해하는 데 도움이 되는 배경 개념이면 슬라이드로 추가

## 6. 산출물 형태

- 슬라이드 여러 장을 위 형식대로 순서대로 나열하여 텍스트로 작성
- 슬라이드 간에는 구분선(`---`) 등으로 명확히 구분
